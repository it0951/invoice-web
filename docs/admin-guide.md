# 관리자 사용 가이드

invoice-web 서비스의 관리자가 알아야 할 핵심 기능과 조작 방법을 설명합니다.

---

## 1. 로그인 방법

1. 브라우저에서 서비스 URL로 접속합니다 (예: `https://your-domain.com`).
2. 로그인 페이지(`/login`)로 자동 이동합니다.
3. 환경변수에 설정한 **관리자 이메일**과 **비밀번호**를 입력합니다.
4. **"로그인"** 버튼을 클릭합니다.
5. 로그인 성공 시 대시보드(`/dashboard`)로 이동합니다.

> 로그인 정보는 `ADMIN_EMAIL` / `ADMIN_PASSWORD_HASH` 환경변수로 관리됩니다.  
> 비밀번호를 변경하려면 [bcrypt 해시 생성](#6-bcrypt-비밀번호-해시-생성-명령어) 후 환경변수를 업데이트하고 재배포해야 합니다.

---

## 2. 대시보드 사용법

로그인 후 진입하는 `/dashboard` 페이지에서 견적서 목록을 확인할 수 있습니다.

### 견적서 목록 테이블

| 컬럼 | 설명 |
|------|------|
| 제목 | 견적서 제목 (Notion DB의 `title` 속성) |
| 고객명 | 발행 대상 고객 이름 |
| 발행일 | 견적서 발행일 |
| 상태 | 초안 / 발송 / 확정 중 하나 |
| 총액 | 견적서 합계 금액 |
| 공유 상태 | 링크 없음 / 활성 / 만료 / 회수 |
| 조회수 | 고객이 공유 링크를 열람한 횟수 |
| 작업 | 공유 링크 발급 / 회수 버튼 |

- 목록은 발행일 기준 최신순으로 정렬됩니다.
- 데이터는 60초 간격으로 캐시가 갱신됩니다. 즉시 최신 데이터를 보려면 페이지를 새로고침하세요.

---

## 3. 공유 링크 발급 방법

고객에게 견적서를 공유하려면 공유 링크(토큰 기반 URL)를 발급해야 합니다.

1. 대시보드의 견적서 목록에서 해당 견적서 행 우측의 **"공유 링크 발급"** 버튼을 클릭합니다.
2. 공유 설정 다이얼로그가 열립니다.
3. **만료일**을 선택합니다 (7일 / 30일 / 90일 중 선택).
4. **"링크 생성"** 버튼을 클릭합니다.
5. 생성된 링크가 표시되면 **"복사"** 버튼으로 클립보드에 복사합니다.
6. 복사한 링크를 이메일이나 메신저로 고객에게 전달합니다.

> 생성된 링크 형식: `https://your-domain.com/invoice/[UUID 토큰]`  
> 만료일이 지나면 고객이 링크에 접속해도 만료 안내 페이지(`/error`)로 이동합니다.

---

## 4. 공유 링크 회수 방법

이미 발급한 공유 링크를 즉시 무효화해야 하는 경우 회수 기능을 사용합니다.

1. 대시보드에서 해당 견적서 행 우측의 **"링크 회수"** 버튼을 클릭합니다.
2. 확인 다이얼로그에서 **"회수"** 버튼을 클릭합니다.
3. 회수 완료 후 해당 링크로 접속하면 고객은 회수 안내 페이지로 이동합니다.

> 회수한 링크는 복구할 수 없습니다. 필요 시 새 링크를 다시 발급해야 합니다.  
> 회수 처리 시 Notion DB의 `tokenRevokedAt` 필드에 회수 시각이 기록됩니다.

---

## 5. Notion에서 견적서 데이터 입력 방법

견적서 데이터는 Notion DB에 직접 입력합니다. 이 서비스는 Notion을 읽기 전용 데이터 소스로 사용합니다.

### Invoice DB 입력 순서

1. Notion에서 **Invoice DB** 페이지를 엽니다.
2. **"+ New"** 버튼을 클릭하여 새 행(견적서)을 추가합니다.
3. 아래 속성을 입력합니다.

| 속성명 | 입력 내용 |
|--------|-----------|
| `title` (Aa, 기본 제목) | 견적서 이름 (예: "웹사이트 개발 견적서") |
| `clientName` | 고객 이름 |
| `clientEmail` | 고객 이메일 주소 |
| `issueDate` | 발행일 (Date 형식) |
| `status` | 초안 / 발송 / 확정 중 선택 |
| `totalAmount` | 총 견적 금액 (숫자) |

> `shareToken`, `tokenExpiresAt`, `tokenRevokedAt`, `viewCount`, `lastViewedAt` 속성은 이 서비스가 자동으로 관리합니다. 직접 수정하지 마세요.

### InvoiceItem DB 입력 방법 (항목 추가)

각 견적서에 항목(라인 아이템)을 추가합니다.

1. **InvoiceItem DB** 페이지를 엽니다.
2. **"+ New"** 버튼으로 새 항목을 추가합니다.
3. 아래 속성을 입력합니다.

| 속성명 | 입력 내용 |
|--------|-----------|
| `title` (Aa, 기본 제목) | 항목 설명 (예: "프론트엔드 개발") |
| `quantity` | 수량 (숫자) |
| `unitPrice` | 단가 (원, 숫자) |
| `subtotal` | 소계 (선택 사항 — 코드가 quantity × unitPrice로 자동 계산) |
| `invoiceId` (Relation) | 상위 Invoice DB의 견적서 선택 |

### InvoiceItem — invoiceId Relation 설정 방법

`invoiceId` 필드는 상위 견적서와의 연결(Relation) 속성입니다.

1. InvoiceItem의 `invoiceId` 셀을 클릭합니다.
2. 팝업에서 연결할 Invoice DB의 견적서 이름을 검색합니다.
3. 해당 견적서를 선택하여 Relation을 연결합니다.

> Relation이 올바르게 설정되지 않으면 해당 항목이 견적서 공개 페이지에 표시되지 않습니다.

## 6. bcrypt 비밀번호 해시 생성 명령어

관리자 비밀번호를 변경하거나 초기 설정할 때 bcrypt 해시를 생성합니다.

```bash
node -e "const bcrypt = require('bcrypt'); bcrypt.hash('YOUR_PASSWORD', 12).then(h => console.log(h))"
```

- `YOUR_PASSWORD` 자리에 설정할 비밀번호를 입력합니다.
- 출력된 문자열(`$2b$12$...`)을 `ADMIN_PASSWORD_HASH` 환경변수에 설정합니다.
- 변경 후 서비스를 재배포해야 반영됩니다.

> bcrypt 패키지가 없는 경우 `npm install bcrypt`로 설치 후 실행하세요.
